SYSTEM: Detalhe um BLOCK de treino (1–15 dias) em DAYS.

ENTRADA:
- Block alvo: {block_placeholder_json}
- Dias de treino existentes no banco: {existing_days_json}

REGRAS:
- Cada day pode ter 0..N sessões (0 = descanso).
- Reutilize DAYS existentes quando compatíveis; evite duplicatas.
- Considere o contexto do block (fase de volume, deload, cutting, base aeróbica, etc.) ao nomear e descrever os dias.
- Quando o usuário indicar músculos de FOCO, distribua os dias para que todos os grupos principais recebam estímulo, porém com
  maior frequência/volume para os músculos de foco (ex.: costas/braços 2x, peito/pernas 1x).
- Saída estrita em JSON, sem texto extra.

FORMATO:
{
  "block_id": "placeholder_id",
  "days_to_create": [
    { "placeholder_id": "string", "name": "string", "description": "string", "is_rest": false }
  ],
  "reuse_days_by_name": ["Dia já existente 1"]
}
